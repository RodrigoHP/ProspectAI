# Story 1.2 — Validação de inputs com Zod + react-hook-form

## Status: Ready for Review

## Story

**Como** usuário do ProspectAI,
**Quero** que o formulário de busca valide adequadamente os campos antes de enviar,
**Para que** requisições inválidas ou maliciosas não cheguem à API Gemini.

## Story Context

**Existing System Integration:**
- Integra com: `components/SearchForm.tsx`
- Technology: React 19 · TypeScript · `@hookform/resolvers` (já instalado) · Zod (instalar)
- Follows pattern: Componentes de form existentes com handlers onChange/onSubmit
- Touch points: Props `onSearch(params: SearchParams)` e `isLoading: boolean`

**Débitos resolvidos:**
- [P1] D5 — Inputs sem validação real (apenas `required` HTML)
- [P2] A6 — `@hookform/resolvers` instalado sem `react-hook-form`

## Technical Notes

- **Integration Approach:** `react-hook-form` gerencia o estado interno do form; `onSearch` só é chamado após validação OK
- **Existing Pattern Reference:** Componente atual usa `useState` por campo — migrar para `register`/`handleSubmit`
- **Key Constraints:**
  - Manter o `<select>` de estado funcionando (registrar com `register("state")`)
  - `@hookform/resolvers` já instalado — apenas instalar `react-hook-form` e `zod`
  - Não alterar a interface pública do componente (`onSearch`, `isLoading`)

## Dev Notes

```typescript
// lib/validations.ts
import { z } from "zod"
export const SearchParamsSchema = z.object({
  icp: z.string().min(10, "Mínimo 10 caracteres").max(500).trim(),
  service: z.string().min(10, "Mínimo 10 caracteres").max(500).trim(),
  state: z.string().min(1, "Selecione um estado"),
  city: z.string().max(100).trim().optional().default(""),
})
```
