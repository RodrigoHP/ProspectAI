# Story 2.3 — Limpeza de dependências e tech debt de configuração

## Status: Draft

## Story

**Como** desenvolvedor do ProspectAI,
**Quero** remover dependências não utilizadas e corrigir inconsistências de configuração,
**Para que** o projeto tenha um `package.json` limpo, builds mais rápidos e sem código morto.

## Story Context

**Existing System Integration:**
- Integra com: `package.json`, `hooks/use-mobile.ts`, `types/index.ts`
- Technology: Node.js · npm · TypeScript
- Follows pattern: Projeto enxuto sem dependências desnecessárias
- Touch points: `package.json` deps, campo `types` em `Lead`, hook `use-mobile`

**Débitos resolvidos:**
- [P3] A8 — `firebase-tools` em devDeps sem nenhum uso
- [P3] A9 — `motion` (Framer Motion) instalado mas sem uso real
- [P3] A10 — `package.json name` = `"ai-studio-applet"` diverge do nome real
- [P3] F6 — `hooks/use-mobile.ts` definido mas não usado por nenhum componente
- [P2] D8 — Campo `types: string[]` no tipo Lead nunca populado pelo LLM
