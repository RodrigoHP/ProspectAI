## Technical Notes

- **Integration Approach:** Vitest com jsdom emula o browser; Testing Library renderiza componentes React
- **Existing Pattern Reference:** Sem padrão existente — criar estrutura de `__tests__/` ou arquivos `.test.ts` colocalizados
- **Key Constraints:**
  - Next.js 15 com App Router requer configuração específica no vitest.config (aliases `@/*`)
  - Componentes com `"use client"` funcionam normalmente no jsdom
  - Mockar fetch nos testes de SearchForm (não chamar API real)

## Dev Notes

```typescript
// vitest.config.ts básico
import { defineConfig } from 'vitest/config'
import react from '@vitejs/plugin-react'
import path from 'path'

export default defineConfig({
  plugins: [react()],
  test: {
    environment: 'jsdom',
    setupFiles: ['./vitest.setup.ts'],
    globals: true,
  },
  resolve: {
    alias: { '@': path.resolve(__dirname, '.') }
  }
})
```

- **Primary Risk:** Conflitos de configuração entre Next.js e Vitest no resolve de paths
