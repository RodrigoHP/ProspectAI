# Story 2.2 — Empty state + Acessibilidade básica

## Status: Draft

## Story

**Como** usuário do ProspectAI,
**Quero** feedback visual claro quando a busca não retorna resultados e que alertas de erro sejam anunciados por leitores de tela,
**Para que** a experiência seja inclusiva e intuitiva em todos os cenários de uso.

## Story Context

**Existing System Integration:**
- Integra com: `components/ResultsList.tsx`, `app/page.tsx`
- Technology: React 19 · Tailwind 4 · Lucide React
- Follows pattern: Design system existente (slate/blue/rose palette)
- Touch points: `{results.length} Leads Encontrados` e banner de erro em `app/page.tsx:149`

**Débitos resolvidos:**
- [P2] F2 — Sem empty state quando busca retorna 0 leads
- [P2] F3 — Sem `role="alert"` no banner de erro
- [P2] F4 — Spinner sem `aria-label`
- [P2] F5 — `<select>` nativo fora do design system

## Acceptance Criteria

### Empty State
- [ ] AC1: Quando `results.length === 0`, exibir estado vazio em vez da grid/tabela vazia
- [ ] AC2: Empty state contém: ícone ilustrativo, título "Nenhum lead encontrado", subtítulo sugerindo ajustar os parâmetros de busca, e botão "Nova Busca"
- [ ] AC3: Empty state segue a paleta existente (slate/blue) e tem visual consistente com o restante da UI
- [ ] AC4: Clicar em "Nova Busca" no empty state chama `onBack()`

### Acessibilidade
- [ ] AC5: Banner de erro em `app/page.tsx` recebe `role="alert"` e `aria-live="polite"`
- [ ] AC6: Spinner de loading em `LeadDetail.tsx` recebe `aria-label="Gerando relatório..."` e `role="status"`
- [ ] AC7: Spinner de loading em `SearchForm.tsx` (botão) recebe `aria-label` adequado quando em loading

### Select — Design System
- [ ] AC8: `<select>` nativo de estado em `SearchForm.tsx` substituído por elemento com classes visuais consistentes com o design system (mesmo visual do `Input` component)

### Regressão
- [ ] AC9: Quando resultados existem, grid/tabela exibidos normalmente
- [ ] AC10: Comportamento de back e seleção de lead não alterados

## Technical Notes

- **Integration Approach:** Mudanças localizadas em 2 componentes + page principal — sem novos componentes necessários
- **Existing Pattern Reference:** Padrão de error display já existe em `app/page.tsx:149` — seguir o mesmo estilo visual
- **Key Constraints:**
  - Empty state deve aparecer quando `results.length === 0` — não quando `isLoading` está true
  - Não instalar bibliotecas de acessibilidade — usar atributos HTML nativos
  - O `<select>` pode manter a tag `select` mas deve receber classes Tailwind do design system

## Dev Notes

    <Button onClick={onBack}>Nova Busca</Button>
