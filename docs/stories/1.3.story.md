# Story 1.3 — Corrigir error recovery em LeadDetail (sem reload de página)

## Status: Ready for Review

## Story

**Como** usuário que está visualizando detalhes de um lead,
**Quero** que erros na geração de relatório permitam tentar novamente sem recarregar a página,
**Para que** não perca os resultados de busca e o contexto atual ao clicar em "Tentar Novamente".

## Story Context

**Existing System Integration:**
- Integra com: `components/LeadDetail.tsx`
- Technology: React 19 · useState · useEffect
- Follows pattern: Padrão de estado `isLoading / error / data` já existente no componente
- Touch points: Botão "Tentar Novamente" na linha 243 de `LeadDetail.tsx`

**Débito resolvido:**
- [P1] F1 — `window.location.reload()` destrói todo o estado do usuário

## Acceptance Criteria

### Funcional
- [x] AC1: Botão "Tentar Novamente" **não** executa `window.location.reload()`
- [x] AC2: Ao clicar "Tentar Novamente", a função `fetchReport` é reexecutada (nova chamada à API)
- [x] AC3: Estado de loading (`isLoading: true`) exibido durante o retry
- [x] AC4: Após retry bem-sucedido, relatório é exibido normalmente
- [x] AC5: Usuário permanece na página de detalhe do lead — resultados anteriores não são perdidos

### UX
- [x] AC6: Botão "Tentar Novamente" exibe estado de loading durante retry (disabled + texto "Tentando...")

### Regressão
- [ ] AC7: Fluxo normal (sem erro) continua funcionando identicamente
- [ ] AC8: Botão "Voltar à Lista" continua funcionando após retry

## Technical Notes

- **Integration Approach:** Extrair `fetchReport` para fora do `useEffect` como função separada, chamar no `useEffect` inicial e no retry
- **Existing Pattern Reference:** `isLoading` / `error` / `report` states já existem — apenas reconectar o retry a eles
- **Key Constraints:**
  - Após a story 1.1, `fetchReport` chamará `fetch('/api/report', ...)` — este retry deve fazer o mesmo
  - Não alterar a interface de props do componente (`lead`, `searchParams`, `onBack`)

## Dev Notes

```typescript
// Padrão esperado — extrair fetchReport como função:
const fetchReport = async () => {
  setError(null)
  setIsLoading(true)
  try {
    // fetch('/api/report', ...) — após story 1.1
    ...
  } catch (err) { ... }
  finally { setIsLoading(false) }
}

useEffect(() => { fetchReport() }, [lead, searchParams])

// Botão retry:
<Button onClick={fetchReport} disabled={isLoading}>
  {isLoading ? "Tentando..." : "Tentar Novamente"}
</Button>
```

- **Primary Risk:** Minimal — mudança localizada em um componente
- **Mitigation:** Testar: erro → retry → sucesso; erro → retry → erro novamente
- **Rollback:** Reverter `LeadDetail.tsx`

**Nota:** Esta story deve ser implementada **após** a story 1.1, pois `fetchReport` deve chamar o Route Handler, não o SDK diretamente.
