# Story 1.1 — Mover chamadas Gemini para Route Handlers (Segurança P0)

## Status: Ready for Review

## Story

**Como** desenvolvedor/operador do ProspectAI,
**Quero** que as chamadas à API Gemini sejam feitas exclusivamente no servidor (Next.js Route Handlers),
**Para que** a chave de API nunca seja exposta no browser e o modelo correto seja utilizado em produção.

## Story Context

**Existing System Integration:**
- Integra com: `app/page.tsx` (busca de leads) e `components/LeadDetail.tsx` (geração de relatório)
- Technology: Next.js 15 App Router · `@google/genai` · TypeScript
- Follows pattern: Next.js Route Handlers em `app/api/`
- Touch points: `process.env.NEXT_PUBLIC_GEMINI_API_KEY` → `process.env.GEMINI_API_KEY`

**Débitos resolvidos nesta story:**
- [P0] A1 — API Key Gemini exposta via `NEXT_PUBLIC_`
- [P0] D2 — Modelo `gemini-3.1-pro-preview` inexistente
- [P1] D3 — Sem backend API layer
- [P1] D4 — JSON parsing frágil com fallback regex

## Technical Notes

- **Integration Approach:** Route Handlers Next.js ficam em `app/api/{endpoint}/route.ts`, exportam função `POST`
- **Existing Pattern Reference:** App Router Route Handlers — `export async function POST(request: Request)`
- **Key Constraints:**
  - `GEMINI_API_KEY` deve ser lida **apenas no servidor** — nunca prefixar com `NEXT_PUBLIC_`
  - Manter mesmo shape de resposta para não quebrar componentes cliente
  - Usar `tools: [{ googleMaps: {} }]` na busca (mantém integração Maps)
  - Modelo correto para relatório: `gemini-2.5-flash`
