# Story 2.1 — CI/CD: GitHub Actions + ESLint no build

## Status: Draft

## Story

**Como** desenvolvedor do ProspectAI,
**Quero** um pipeline de CI automatizado e ESLint ativo no build,
**Para que** regressões de lint e type errors sejam bloqueados antes de chegar à main.

## Story Context

**Existing System Integration:**
- Integra com: `.github/workflows/` (criar), `next.config.ts`, `package.json`
- Technology: GitHub Actions · ESLint 9 · TypeScript 5.9 · Next.js 15
- Follows pattern: Projetos Next.js com workflow de CI padrão
- Touch points: `next.config.ts` linha 5-7 (`eslint.ignoreDuringBuilds`)

**Débitos resolvidos:**
- [P2] A4 — `eslint: { ignoreDuringBuilds: true }` silencia erros no deploy
- [P2] A7 — Sem `.github/workflows/` — zero CI automatizado

## Acceptance Criteria

### ESLint no build
- [ ] AC1: `next.config.ts` alterado: `eslint.ignoreDuringBuilds: false`
- [ ] AC2: `npm run build` continua passando após a mudança (sem erros de lint ativos)
- [ ] AC3: `npm run lint` passa sem erros

### GitHub Actions
- [ ] AC4: Criar `.github/workflows/ci.yml` que executa em `push` e `pull_request` na branch `main`
- [ ] AC5: Workflow executa em sequência: `npm ci` → `npm run lint` → `npm run build`
- [ ] AC6: Se testes estiverem configurados (story 1.4 concluída), adicionar step `npm run test:run`
- [ ] AC7: Job usa `ubuntu-latest` e `node-version: 20`
- [ ] AC8: Cache de `node_modules` configurado para acelerar runs

### Regressão
- [ ] AC9: `npm run build` local passa após ativar ESLint
- [ ] AC10: Workflow não falha em condições normais (push na main com código atual)

## Technical Notes

- **Integration Approach:** Workflow simples de CI — sem deploy (Vercel já faz deploy automaticamente)
- **Existing Pattern Reference:** `.github/workflows/` padrão Next.js
- **Key Constraints:**
  - Verificar se há erros de lint existentes **antes** de ativar `ignoreDuringBuilds: false` — corrigi-los primeiro
  - Não configurar secrets de deploy no workflow (Vercel usa integração GitHub nativa)

## Dev Notes

```yaml
# .github/workflows/ci.yml básico
name: CI
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
jobs:
  ci:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - run: npm ci
      - run: npm run lint
      - run: npm run build
      # - run: npm run test:run  # descomentar após story 1.4
```

## Risk & Compatibility

- **Primary Risk:** Erros de lint latentes quebram o build após ativar ESLint
- **Mitigation:** Executar `npm run lint` **antes** de alterar a config e corrigir todos os erros encontrados
- **Rollback:** Reverter `ignoreDuringBuilds: true` temporariamente se necessário
